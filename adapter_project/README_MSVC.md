# Инструкция по компиляции и запуску в PowerShell с MSVC

## Вариант 22 - Обработка бинарного файла adapter.dat

### Требования
- Visual Studio с установленным компонентом "C++ desktop development"
- PowerShell
- Командная строка разработчика Visual Studio (Developer Command Prompt)

---

## Шаг 1: Откройте командную строку разработчика

**Вариант А:** Через меню Пуск
1. Нажмите Пуск
2. Найдите "Developer Command Prompt for VS 2022" (или ваша версия)
3. Запустите от имени администратора или обычного пользователя

**Вариант Б:** Через PowerShell
```powershell
# Найдите путь к vcvarsall.bat (путь может отличаться в зависимости от версии)
$vsPath = & "${env:ProgramFiles(x86)}\Microsoft Visual Studio\Installer\vswhere.exe" -latest -products * -requires Microsoft.Component.MSBuild -find installationPath
if ($vsPath) {
    & "$vsPath\Common7\Tools\VsDevCmd.bat"
}
```

## Шаг 3: Компиляция программы создания файла

```powershell
cl /EHsc /O2 /utf-8 create_adapter.cpp /Fe:create_adapter.exe
```

**Параметры компилятора:**
- `/EHsc` - обработка исключений C++
- `/O2` - оптимизация кода
- `/utf-8` - поддержка UTF-8 для русской локали
- `/Fe:` - имя выходного файла

---

## Шаг 4: Компиляция программы обработки файла

```powershell
cl /EHsc /O2 /utf-8 process_adapter.cpp /Fe:process_adapter.exe
```

---

## Шаг 5: Запуск программ

### 5.1 Создание бинарного файла

```powershell
.\create_adapter.exe
```

**Ожидаемый результат:**
```
Файл adapter.dat успешно создан!
Количество записей: 20
Размер одной записи: 184 байт
Общий размер файла: 3680 байт
```

### 5.2 Запуск программы обработки

```powershell
.\process_adapter.exe
```

**Программа откроет меню с опциями:**
1. Показать все записи
2. Поиск по типу адаптера
3. Редактировать запись
4. Сохранить изменения
5. Информация о файле
6. Информация о разработчике
0. Выход

## Устранение ошибок

### Ошибка: "cl" не является внутренней или внешней командой

**Решение:** Вы не запустили Developer Command Prompt. Используйте шаг 1.

### Ошибка: Не удается открыть файл adapter.dat

**Решение:** Сначала запустите `create_adapter.exe` для создания файла.

### Ошибка кодировки русского текста

**Решение:** Убедитесь, что используете параметр `/utf-8` при компиляции.

### Ошибка линковки (LNK2019)

**Решение:** Проверьте, что все библиотеки подключены. Для данного проекта дополнительные библиотеки не требуются.

## Быстрая компиляция (один скрипт)

Создайте файл `build.ps1`:

```powershell
Write-Host "Компиляция create_adapter.cpp..." -ForegroundColor Green
cl /EHsc /O2 /utf-8 create_adapter.cpp /Fe:create_adapter.exe

Write-Host "Компиляция process_adapter.cpp..." -ForegroundColor Green
cl /EHsc /O2 /utf-8 process_adapter.cpp /Fe:process_adapter.exe

Write-Host "Сборка завершена!" -ForegroundColor Green
Write-Host "Запустите .\create_adapter.exe для создания файла данных" -ForegroundColor Yellow
```

Запуск:
```powershell
.\build.ps1
```
