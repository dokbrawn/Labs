# Backend на C++ для каталога Library

Этот модуль покрывает backend-часть задачи из требований:
- хранение книг/жанров/поджанров на диске;
- сортировка по любому полю;
- бинарный поиск (сначала по названию, затем по автору);
- построение дерева оптимального поиска (optimal BST) по ISBN;
- подгрузка данных из сети при добавлении записи;
- хранение ссылок на изображения обложки/лицензии для каждой книги.

## Почему выбраны такие решения

1. **C++17 + CMake**
   - легко собрать на x86 Linux/Windows;
   - нет тяжелых внешних зависимостей;
   - удобно встроить как backend-слой для GUI (Qt, ImGui, Electron+native module и т.д.).

2. **TSV-файл как хранилище (`data/library.tsv`)**
   - требование "хранить в файле на диске" выполнено напрямую;
   - формат легко читать/отлаживать вручную;
   - подходит для учебной задачи без персональных данных.

3. **Иерархия жанр → поджанр реализована в модели `Book`**
   - поля `genre` и `subgenre` позволяют строить дерево в UI;
   - backend не привязан к конкретному интерфейсу.

4. **Сортировка по всем полям через `SortField`**
   - одно API `sortedBooks(field, ascending)`;
   - удобно подключать любой вид сортировки из GUI.

5. **Бинарный поиск для сценария "сначала название, потом автор"**
   - `searchBooks()` сначала сортирует по `title` и применяет `lower_bound`;
   - если не найдено, повторяет по `author`.

6. **Optimal BST по ISBN**
   - метод `buildOptimalSearchTreeByIsbn()` считает DP-таблицы `cost/root`;
   - поле `searchFrequency` задает частоты обращений;
   - итог можно использовать как ускоренную структуру поиска для "горячих" книг.

7. **Сетевое подтягивание данных**
   - `NetworkMetadataClient` при добавлении книги запрашивает OpenLibrary по ISBN;
   - если локально не задано название, backend пытается подставить из сети.

## Сборка и запуск

```bash
cd library_backend_cpp
cmake -S . -B build
cmake --build build
./build/library_backend
```

## Интеграция с GUI

GUI может вызывать сервисные методы:
- `addBook(...)`, `removeBookById(...)`
- `searchBooks(query)`
- `sortedBooks(field, ascending)`
- `buildOptimalSearchTreeByIsbn()`

Так backend остается универсальным, а графический слой можно сделать на любом стеке.
